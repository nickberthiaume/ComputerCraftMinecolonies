# Workspace Assistant Instructions

These instructions apply to all prompts in this repository.

- When modifying a file, include or update a `version` field in that file.
- Store the version as a string in the form `vX.Y`, where `X` is the major version and `Y` is the minor version.
- For normal incremental changes, bump the version by `0.1`.
  - Example: `v1.0` → `v1.1` → `v1.2`
- If a whole-number major version is specifically requested or appropriate, use a whole number format such as `v2.0`.
- Treat the version string as representing a numerical value, so `v1.10` is valid and expresses a minor version after `v1.9`.
- If the file already has a `version` field, update it consistently according to these rules.
- If the file type supports comments or metadata, place the `version` field in a natural location near the top of the file.
