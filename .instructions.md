# Workspace Assistant Instructions

These instructions apply to all prompts in this repository.

- Every file edit must include or update a `version` field in that file. This is mandatory for every assistant-generated code change.
- Store the version as a string in the form `vX.Y`, where `X` is the major version and `Y` is the minor version.
- For normal incremental changes, bump the minor version by `0.1`.
  - Example: `v1.0` → `v1.1` → `v1.2`
- For larger or breaking changes, bump the major version and reset the minor version to `0`.
  - Example: `v1.2` → `v2.0`
- Treat the version string as representing a numerical value, so `v1.10` is valid and represents a later version than `v1.9`.
- If the file already has a `version` field, update it consistently whenever the file is edited.
- If a file has no version field, add one near the top of the file before committing changes.
- When adding code that draws panels or filled boxes, clear the target area first and avoid nested screen launches. Screen modules should return control to the caller instead of calling `shell.run("main")` from inside a screen.
- If the file type supports comments or metadata, place the `version` field in a natural location near the top of the file.
